🌳 Git Feature Branch 工作流

创建分支 → 开发提交 → 合并回主分支 → 查看分支图

📋 操作步骤演示

# Step 1: 创建并切换到功能分支
$ git checkout -b feature/gis-demo
Switched to a new branch 'feature/gis-demo'

# Step 2: 在功能分支上开发并提交
$ git add .
$ git commit -m "feat: 初始化 GIS 项目结构"
[feature/gis-demo e87405c] feat: 初始化 GIS 项目结构
 36 files changed, 4047 insertions(+)

$ git commit -m "docs: 更新 README，增加开发分支说明"
[feature/gis-demo 8f3a2b1] docs: 更新 README，增加开发分支说明
 1 file changed, 5 insertions(+)

# Step 3: 切换回 main 分支
$ git checkout main
Switched to branch 'main'

# Step 4: 合并功能分支
$ git merge feature/gis-demo -m "merge: 合并 feature/gis-demo 分支"
Merge made by the 'ort' strategy.
 37 files changed, 4052 insertions(+)

# Step 5: 查看分支图
$ git log --graph --oneline

✅ 工作流总结

Feature Branch 工作流优点

  • 功能开发与主线隔离，互不影响
  • 方便代码审查和回滚
  • 保持 main 分支始终可用
  • 支持并行开发多个功能
  • 提交历史清晰可追溯

💡 常用 Git 分支命令

# 分支操作
git branch                    # 查看本地分支
git branch -a                 # 查看所有分支
git checkout -b feature/xxx   # 创建并切换到新分支
git checkout main             # 切换到 main 分支
git branch -d feature/xxx     # 删除分支

# 合并操作
git merge feature/xxx          # 合并指定分支到当前分支
git merge --no-ff feature/xxx # 非快速合并，保留分支历史

# 查看历史
git log --oneline             # 简洁提交历史
git log --graph               # 分支图
git log --graph --oneline      # 简洁分支图
